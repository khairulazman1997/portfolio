t/Place

An Online Collaborative Canvas for Tembusu Welcome Week

Previous Next

t/Place is an online collaborative canvas that is highly inspired by Reddit’s April’s Fool event, r/Place. Participants are only allowed to draw a pixel at a time and pixels are given once every 5 minutes so collaboration is required to draw anything substantial. t/Place ran during the duration of Tembusu Welcome Week as part of an e-orientation from 1st to 3rd August 2020. The final canvas can be seen on the second image above.

As the project lead in a team of 6, I was in charge of maintaining the Express Server as well as telegram bot integration for authentication. To access the web-app, a user has to enter through the bot in a telegram group that has previously been whitelisted. The Express Server works with a PostgreSQL database for user storage as well as a separate Redis server that acts as a cache for the canvas. To ensure the server loads smoothly, we used a bitfield to store the overall canvas that is maintained on the Redis server. Finally we used Socket.io to ensure all the web-app clients (built with vanilla JS) are updated whenever a new pixel is placed.

  • Github Link: https://github.com/Tancho-Welcome-Week/tPlace

Marathon Timer

A timer built for Tembusu Vertical Marathon 2020

Previous Next

Over the course of my Winter Break in AY2019-2020, I was asked to build a timing system for Tembusu’s Vertical Marathon 2020. The previous system was built on Google Sheets which caused a significant difference between actual and recorded time. To prevent any loss of data due to server crashes, we decided to build an entirely offline timer that’s is separated into 4 parts: Registration, Start, End and Complete with a person manning each station. This is to be built with the Address Book as a starting point meaning it is built entirely in Java with JavaFX library for the UI.

At the Registration station, participants are given their bib number which is then recorded along with their names and category in the system. The Start and End station then records the time in which the participant starts and end the marathon respectively. Afterwards the participant will then return their bib numbers as well as the position in which they ended their run to be put in the system in the Complete station. After the end of the event, the data file outputted from the 4 stations will be sent through telegram to us for the final computation of results based on categories.

Barring a few hiccups in calculation due to our relative inexperience with handling data in excel, the timer was able to handle the timing of the marathon well, serving a total of 300+ participants. Beyond just coding, we had to know our timer inside out as we were also tasked to train the station managers on how to use the various aspects of the software to ensure the proper results are recorded.

  • Github Link: https://github.com/vukoh/TVM-Timer
